Automated Support System

Product: MiraTrust Document Type: Feature Overview Date: April 20, 2026 Version: 1.0

Goal: Resolve 99% of user support issues automatically — without hiring support staff. The system operates in three progressive layers, each only activating if the previous layer could not resolve the problem.

1. System Architecture — Three Layers

Every support issue a user faces is routed through three layers in order. A user only reaches the next layer if the current one fails to resolve their issue.

User encounters a problem │ ▼ ┌─────────────────────────────────┐ │ LAYER 1: Self-Healing Errors │ ← App explains and fixes silently └─────────────────────────────────┘ │ Not resolved ▼ ┌─────────────────────────────────┐ │ LAYER 2: AI Chat Widget │ ← Claude AI answers instantly └─────────────────────────────────┘ │ Not resolved ▼ ┌─────────────────────────────────┐ │ LAYER 3: Human Ticket │ ← Team follows up by email └─────────────────────────────────┘

2. Layer 1 — Self-Healing Errors

Fully Automated
App diagnoses and explains problems before the user asks for help

Whenever an API call fails anywhere in the application, the system maps the technical error code to a plain-English message with a direct action button. Users never see raw error codes.

Additionally, a global interceptor watches all API responses. If any request returns a 401 (session expired), it automatically clears the user's session and redirects them to the login page — no component needs to handle this individually.

Error Mapping Examples

Error Condition What the User Sees Action Offered
Session expired (401) Automatically redirected to login Sign In
Out of credits (402) "You've run out of credits." Add Credits → /pricing
No permission (403) "You don't have permission to perform this action." —
Too many requests (429) "You're going too fast. Wait a moment and try again." —
Server error (500) "Something went wrong on our end. We've been notified." —
No internet connection "Can't reach the server. Check your internet connection." —
Insufficient credits (domain code) "You need more credits to issue QR codes." Add Credits → /pricing
QR limit reached (domain code) "You've reached your QR code limit. Upgrade to issue more." View Plans → /pricing

Business Impact: The most common user frustrations — running out of credits and session expiry — are resolved in a single click, with zero support interaction required.

3. Layer 2 — AI Chat Widget

AI-Powered · 24/7
Instant answers to any question via a floating chat panel

A help button is permanently visible in the bottom-right corner of every authenticated page. Users can ask any question and receive an instant, accurate response powered by Anthropic's Claude AI model.

User Experience Flow

  1. User clicks the floating help button (bottom-right corner)
  2. A chat panel opens with a welcome message and four quick-topic shortcuts
  3. User clicks a topic or types a custom question
  4. AI responds within 2–3 seconds
  5. User sees "Was this helpful?" (👍 / 👎) after each response
  6. If 👍 — conversation can continue or be closed
  7. If 👎 — escalation to Layer 3 is offered automatically

Quick Topic Shortcuts

  • How do credits work?
  • I can't generate a QR code
  • Payment or billing issue
  • Verification isn't working

AI Knowledge Base

The AI is configured with a custom MiraTrust knowledge base. It will only answer from this information — it will never invent features, prices, or policies.

Topic Area What the AI Knows
Credits How credits work, pricing tiers ($97 / $499 / $999), that they never expire, how to add more
QR Code Generation Studio section usage, credit consumption, how to troubleshoot generation failures
Verification How to scan, what scan results mean, counterfeit vs. revoked vs. authentic
Billing & Payments How to find transaction history, what to do if credits don't appear after payment
Account & Login Password reset, Google sign-in issues, session management
Common Errors 7 pre-defined issue-and-solution pairs covering the most frequent support requests

Confidence Detection

The system automatically monitors the AI's response for uncertainty. If the AI uses phrases like "I'm not sure", "I don't know", or "contact support", it automatically surfaces the escalation prompt — the user does not need to click 👎 themselves.

Business Impact: Common how-to and billing questions are resolved instantly at any time of day with no wait time and no human involvement.

4. Layer 3 — Human Escalation via Support Ticket

Human-Assisted
Only genuinely complex issues reach a person — with full context already captured

When the AI cannot resolve an issue, it offers to create a support ticket. The entire conversation history is saved automatically so the support agent has complete context before they even send a single reply.

Trigger Conditions

  • User clicks 👎 on an AI response
  • AI detects its own uncertainty (automatic, no user action required)
  • AI backend fails to connect (graceful fallback)

Ticket Creation Flow

  1. Escalation prompt is shown: "Sorry I couldn't fully help. Want to create a support ticket? A team member will follow up by email."
  2. User clicks Create Ticket
  3. System generates a unique ticket reference (e.g. tkt_a3f9b2c1d4)
  4. Full conversation history is saved to the database
  5. User's email is captured automatically from their account
  6. Confirmation is shown: "Ticket created · Ref: tkt_a3f9b2c1d4. We'll follow up by email soon."

What Gets Saved Per Ticket

Field Value Source
Ticket ID Unique reference, e.g. tkt_a3f9b2c1d4 Auto-generated
Original Message User's first question Chat widget
Full Conversation Complete chat history (all messages) Chat widget
User Email Account email address Auto-captured from JWT token
Status open / resolved System
Created At Timestamp (UTC) System

Business Impact: Support agents receive full context before they reply — no back-and-forth needed to understand the problem. Resolution time is significantly reduced.
